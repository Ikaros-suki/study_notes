#### 面向对象方法
* 任何系统由能够完成一组相关任务的==对象==构成
* 调用其他的方法
* 对象之间**不能直接**互相访问内部数据
* 所有交流通过==方法==实现

* 优点:
    1. 模块化
    2. 自然性
    3. 并发性
    4. 重用性

* 三大特性 : 
    1. 封装 : 隐藏对象属性与细节, 安全
    2. 继承 : 子类继承父类, 复用
    3. 多态 : 多个接口, 扩展, 可维护

* 优势 : 解耦, 封装, 复用, 多态
